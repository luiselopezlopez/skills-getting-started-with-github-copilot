# 🗺️ GitHub Copilot Labs — Starting Guide

> A quick-reference guide to help you work through the four GitHub Skills Copilot labs in the right order.

---

## Prerequisites

Before starting any lab, make sure you have:

- [ ] A GitHub account with **GitHub Copilot** enabled (individual, business, or enterprise plan)
- [ ] Permission to create repositories and open issues in your account
- [ ] A supported editor installed — [VS Code](https://code.visualstudio.com/) with the [GitHub Copilot extension](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot) is recommended
- [ ] Basic familiarity with Git (clone, commit, push)

## Lab Summaries & Rationale

### 1 · Getting Started with GitHub Copilot *(start here)*
Learn the fundamentals: how to trigger Copilot suggestions, use inline completions, and work with the Copilot Chat panel. This lab covers the core vocabulary and workflow that every subsequent lab assumes you already know.

### 2 · Build Applications with Copilot Agent Mode
Put Copilot to work on a real feature end-to-end using **agent mode**, where Copilot can run multi-step tasks autonomously. Requires the basics from Lab 1.

### 3 · Expand Your Team with Copilot (Coding Agent)
Explore the **Copilot coding agent** — assign GitHub Issues to Copilot and let it open PRs for you. Builds on the agent-mode concepts practiced in Lab 2.

### 4 · Integrate MCP with Copilot *(most advanced)*
Connect external tools to Copilot via the **Model Context Protocol (MCP)**. This lab assumes you're comfortable with Copilot's agent capabilities from the previous two labs.

## Tips for Success

- **Each lab is self-paced** — take breaks as needed ☕
- **Read the issue instructions carefully** before writing any code; each issue step is a discrete task.
- **Ask Copilot Chat for help** if you get stuck — meta-learning with Copilot is encouraged!
- **Labs are independent repos** — make sure you open the correct repo before starting each exercise.
